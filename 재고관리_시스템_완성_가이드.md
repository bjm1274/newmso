## 개요

병원 ERP 시스템의 재고관리 기능이 완전히 재구성되었습니다. 이제 **UDI 공급내역 보고**, **명세서 자동 작성**, **자동 발주**, **스캔 입고**, **촬영 입고**, **상세 물품 등록** 등 모든 기능이 정상적으로 작동합니다.

### 주요 개선사항
- ✅ UDI 공급내역 보고 대상 자동 구분
- ✅ 거래처 정보 저장 및 명세서 자동 작성
- ✅ 최소 재고 미달 시 자동 발주 및 행정팀 알림
- ✅ 명세서 스캔 시 유효기간/LOT번호 추가 입력 확인
- ✅ 촬영을 통한 제품 정보 자동 인식
- ✅ 상세 물품 등록 (공급업체, 유효기간, LOT번호, UDI 대상 등)

## 주요 기능

### 1. **UDI 관리**
- 의료기기법에 따른 공급내역 보고 대상 품목 관리
- UDI 대상 품목 자동 필터링
- CSV 형식 보고서 자동 생성 및 다운로드
- 품목별 LOT번호, 유효기간 관리

### 2. **명세서 관리**
- 거래처 정보 사전 등록 및 저장
- 명세서 작성 시 거래처 정보 원클릭 불러오기
- 품목 자동완성 (재고 목록에서 검색)
- 인쇄 기능 지원

### 3. **발주 관리**
- 안전재고 미달 품목 자동 감지
- 자동 발주서 생성
- 공급업체별 발주서 그룹화
- 행정팀 알림 자동 전송
- 발주서 승인/취소 관리

### 4. **스캔 입고**
- 바코드 스캔: 제품 정보 자동 인식
- 명세서 스캔: 품목명, 수량, 단가 자동 인식
- **명세서 스캔 시 유효기간/LOT번호 추가 입력 여부 확인 창**
- 일괄 입고 처리

### 5. **촬영 입고**
- 제품 바코드, 제품명, 유효기간, LOT번호 자동 인식
- OCR 기반 텍스트 인식
- 인식 정보 확인 및 수정 기능

### 6. **물품 등록**
- 상세 정보 입력 (제품명, 분류, 제조사, 바코드 등)
- 공급업체 연결
- 유효기간, LOT번호 관리
- **공급내역 보고 대상 (UDI) 여부 선택**
- 최소 재고 수량 설정

### 7. **재고 현황**
- 실시간 재고 현황 조회
- UDI 대상 품목 표시
- 안전재고 미달 품목 강조
- 입출고 간편 처리
- 자동 결재 상신 제안

## 기능별 사용법

### 1. UDI 공급내역 보고

1. **재고관리** 메뉴에서 **📡 UDI** 버튼 클릭
2. UDI 대상 품목 목록 확인
3. 보고할 품목 선택 (전체 선택 가능)
4. **📊 보고서 생성** 버튼 클릭
5. CSV 파일 자동 다운로드
6. 식약처 시스템에 업로드

**주의사항:**
- UDI 대상 품목은 물품 등록 시 "공급내역 보고 대상" 체크박스를 선택해야 합니다.
- LOT번호와 유효기간은 정확히 입력해야 합니다.

---

### 2. 명세서 작성

1. **재고관리** 메뉴에서 **📄 명세서** 버튼 클릭
2. **거래처 관리** 섹션에서 거래처 추가 (필요 시)
3. **📄 명세서 작성** 버튼 클릭
4. 공급자/공급받는자 정보 입력
   - 우측 상단의 버튼으로 저장된 거래처 정보 원클릭 불러오기
5. 품목 입력
   - 품목명 입력 시 자동완성 기능 활용
   - 수량, 단가 입력 시 공급가액 자동 계산
6. **🖨️ 인쇄** 버튼으로 명세서 출력

**팁:**
- 자주 사용하는 거래처 정보는 "💾 업체정보 저장" 버튼으로 저장하면 다음에 쉽게 불러올 수 있습니다.

---

### 3. 자동 발주

1. **재고관리** 메뉴에서 **📝 발주** 버튼 클릭
2. **안전재고 미달 품목** 섹션에서 부족한 품목 확인
3. **🚨 자동 발주 생성** 버튼 클릭
4. 발주서가 자동으로 생성되고 행정팀에 알림 전송
5. **발주서 목록**에서 생성된 발주서 확인
6. 필요 시 승인 또는 취소

**자동화:**
- 재고가 최소 재고 이하로 떨어지면 자동으로 발주 대상으로 표시됩니다.
- 행정팀에 자동으로 알림이 전송됩니다.

---

### 4. 스캔 입고

1. **재고관리** 메뉴에서 **📄 스캔** 버튼 클릭
2. 스캔 모드 선택
   - **📱 바코드 스캔**: 제품 바코드 촬영
   - **📄 명세서 스캔**: 거래 명세서 촬영
3. **📷 카메라 시작** 버튼 클릭
4. 대상 촬영 후 **📸 촬영** 버튼 클릭
5. **명세서 스캔의 경우:**
   - "유효기간과 LOT번호를 추가로 입력하시겠습니까?" 확인 창 표시
   - **✏️ 추가 입력**: 품목별로 유효기간/LOT번호 입력
   - **그냥 등록**: 추가 정보 없이 바로 입고
6. 인식된 정보 확인 후 **✅ 입고 처리** 버튼 클릭

**주의사항:**
- 스캔된 제품이 재고 목록에 없으면 입고가 실패합니다.
- 먼저 **물품등록** 탭에서 제품을 등록하세요.

---

### 5. 촬영 입고

1. **재고관리** 메뉴에서 **📷 촬영** 버튼 클릭
2. **📷 카메라 시작** 버튼 클릭
3. 제품의 바코드, 제품명, 유효기간 등이 보이도록 촬영
4. **📸 촬영** 버튼 클릭
5. AI가 자동으로 정보 인식
6. 인식된 정보 확인
   - 정확하면 **✅ 확인 및 입고**
   - 수정 필요 시 **✏️ 수정하기**
7. 입고 완료

**팁:**
- 조명이 충분하고 글씨가 선명하게 보이도록 촬영하면 인식률이 높아집니다.

---

### 6. 물품 등록

1. **재고관리** 메뉴에서 **+ 등록** 버튼 클릭
2. **기본 정보** 입력
   - 제품명 (필수)
   - 분류 (필수): 의료기기, 소모품, 약품 등
   - 제조사, 바코드, 보관 위치
3. **재고 정보** 입력
   - 현재 수량
   - 최소 재고 (안전재고): 미입력 시 현재 수량의 30%로 자동 설정
   - 단가
4. **공급 정보** 입력
   - 공급 업체 선택
   - LOT 번호
   - 유효기간
   - **공급내역 보고 대상 (UDI)** 체크박스: UDI 대상이면 체크
5. **비고** 입력 (선택)
6. **✅ 물품 등록** 버튼 클릭

**중요:**
- **공급내역 보고 대상 (UDI)** 체크박스를 선택하면 UDI 보고서에 자동으로 포함됩니다.
- 최소 재고를 설정하면 해당 수량 이하로 떨어질 때 자동으로 발주 알림이 전송됩니다.

## 자동화 기능

### 1. 최소 재고 알림 시스템

**작동 방식:**
1. 재고가 최소 재고 수준 이하로 떨어지면 자동 감지
2. 행정팀 사용자에게 알림 자동 전송
3. 발주 리스트에 자동 추가
4. 발주서 자동 생성 (공급업체별 그룹화)

**알림 내용:**
- 제목: "⚠️ 안전재고 미달 알림"
- 내용: "X개 품목의 재고가 최소 수준 이하로 떨어졌습니다. 발주가 필요합니다."
- 메타데이터: 품목 목록, 현재고, 부족량 등

### 2. 자동 결재 상신

**작동 방식:**
1. 재고 현황 화면에서 안전재고 미달 품목 표시
2. "결재 상신" 버튼 클릭
3. 비품구매 신청서 자동 작성
4. 전자결재 시스템에 자동 상신

## 문제 해결

### Q1. 스캔/촬영 시 "등록되지 않은 제품입니다" 오류

**해결방법:**
- 먼저 **물품등록** 탭에서 해당 제품을 등록하세요.
- 제품명이 정확히 일치해야 합니다.

### Q2. UDI 보고서에 품목이 표시되지 않음

**해결방법:**
- 물품 등록 시 **"공급내역 보고 대상 (UDI)"** 체크박스를 선택했는지 확인하세요.
- 기존 품목은 재고 현황에서 수정하여 UDI 대상으로 변경할 수 있습니다.

### Q3. 발주 알림이 전송되지 않음

**해결방법:**
- 최소 재고가 설정되어 있는지 확인하세요.
- 행정팀 사용자가 데이터베이스에 등록되어 있는지 확인하세요 (부서가 '행정부', '총무팀', '원무팀').

### Q4. 명세서 자동완성이 작동하지 않음

**해결방법:**
- 재고 목록에 해당 품목이 등록되어 있는지 확인하세요.
- 품목명을 일부만 입력하면 자동완성 목록이 표시됩니다.

### Q5. 카메라가 작동하지 않음

**해결방법:**
- 브라우저에서 카메라 접근 권한을 허용했는지 확인하세요.
- HTTPS 환경에서만 카메라 접근이 가능합니다.
- 대안: **📁 파일 업로드** 기능을 사용하세요.

## 추가 개발 사항 (선택)

### 1. 실제 OCR 구현

현재는 시뮬레이션 데이터를 사용합니다. 실제 OCR을 구현하려면:

**옵션 1: Tesseract.js (무료)**
```bash
npm install tesseract.js
```

**옵션 2: Google Vision API (유료)**
```bash
npm install @google-cloud/vision
```

**옵션 3: Naver Clova OCR (유료)**
- Naver Cloud Platform에서 API 키 발급

### 2. 바코드 스캔 라이브러리

```bash
npm install quagga
```

### 3. 실시간 알림 (WebSocket)

Supabase Realtime을 활용하여 실시간 알림 구현:

```typescript
const subscription = supabase
  .channel('inventory-alerts')
  .on('postgres_changes', 
    { event: 'INSERT', schema: 'public', table: 'notifications' },
    (payload) => {
      // 실시간 알림 표시
    }
  )
  .subscribe();
```
