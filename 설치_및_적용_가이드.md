## 🚀 설치 단계

### 1단계: 데이터베이스 스키마 업데이트

1. **Supabase 대시보드** 접속
2. **SQL Editor** 메뉴 선택
3. `재고관리_DB_스키마_업데이트.sql` 파일 열기
4. 전체 내용 복사
5. SQL Editor에 붙여넣기
6. **RUN** 버튼 클릭하여 실행
7. 완료 메시지 확인

**예상 실행 시간:** 약 10-20초

---

### 2단계: 파일 교체

#### 방법 1: 기존 파일 백업 후 교체

```bash
# 기존 파일 백업
cd /path/to/your/project
mv main/기능부품/재고관리.tsx main/기능부품/재고관리_백업.tsx

# 새 파일 복사
cp 재고관리_통합완성.tsx main/기능부품/재고관리.tsx
cp -r 재고관리서브/* main/기능부품/재고관리서브/
```

#### 방법 2: 직접 교체

1. 압축 파일 압축 해제
2. `main/기능부품/재고관리_통합완성.tsx` → `main/기능부품/재고관리.tsx`로 이름 변경
3. 기존 `재고관리.tsx` 파일을 새 파일로 교체
4. `재고관리서브/` 폴더의 새 파일들을 기존 폴더에 복사

## 🔧 문제 해결

### 문제 1: "supabase is not defined" 오류

**원인:** Supabase 클라이언트가 설정되지 않음

**해결:**
```typescript
// lib/supabase.ts 파일 확인
import { createClient } from '@supabase/supabase-js';

const supabaseUrl = process.env.NEXT_PUBLIC_SUPABASE_URL!;
const supabaseAnonKey = process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!;

export const supabase = createClient(supabaseUrl, supabaseAnonKey);
```

---

### 문제 2: "Cannot find module" 오류

**원인:** 파일 경로가 잘못됨

**해결:**
- 파일 경로 확인
- import 경로 수정

```typescript
// 상대 경로 확인
import UDIManagement from './재고관리서브/UDI관리';
```

---

### 문제 3: 데이터베이스 테이블이 없음

**원인:** 스키마 업데이트가 실행되지 않음

**해결:**
1. Supabase SQL Editor에서 스키마 업데이트 SQL 재실행
2. 테이블 생성 확인

```sql
-- 테이블 확인
SELECT table_name FROM information_schema.tables 
WHERE table_schema = 'public';
```

---

### 문제 4: 카메라가 작동하지 않음

**원인:** HTTPS 환경이 아니거나 권한 미허용

**해결:**
1. 브라우저에서 카메라 권한 허용
2. HTTPS 환경에서 테스트
3. 대안: 파일 업로드 기능 사용

---

### 문제 5: 알림이 전송되지 않음

**원인:** 행정팀 사용자가 없음

**해결:**
```sql
-- 행정팀 사용자 확인
SELECT * FROM staffs WHERE dept IN ('행정부', '총무팀', '원무팀');

-- 테스트용 행정팀 사용자 추가
INSERT INTO staffs (name, dept, email) VALUES
('관리자', '행정부', 'admin@hospital.com');
```
