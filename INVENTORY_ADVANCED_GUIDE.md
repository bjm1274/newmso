# 재고관리 시스템 고도화 가이드

## 📋 시스템 개요

이 고도화 시스템은 병원의 복잡한 재고 관리 요구사항을 자동화합니다:

1. **UDI 명세서** - 의료기기 공급내역 보고
2. **거래처 명세서** - 거래처 정보 저장 및 명세서 생성
3. **자동 발주** - 안전재고 미달 시 자동 발주서 생성
4. **OCR 스캔** - 명세서 자동 인식 및 일괄 입고
5. **바코드 촬영** - 제품 정보 자동 인식 및 입고
6. **상세 물품등록** - 유효기간, LOT번호, UDI 대상 여부 등 상세 정보 관리

---

## 🎯 주요 기능 설명

### 1. 현황 대시보드
- **전체 품목 수**: 등록된 모든 물품 개수
- **UDI 대상**: 공급내역 보고 대상 의료기기 개수
- **안전재고 미달**: 발주가 필요한 품목 개수
- **거래처**: 등록된 거래처 개수

### 2. UDI 보고
**기능:**
- UDI 대상 품목만 필터링
- 제품명, 수량, LOT번호, 유효기간 표시
- 보고서 자동 생성

**사용 시나리오:**
```
의료기기 공급내역 보고 필요
→ UDI 보고 탭 클릭
→ 보고서 생성 버튼 클릭
→ 자동으로 필요한 정보 수집
→ 보고서 완성
```

### 3. 명세서 (거래처 관리)
**기능:**
- 거래처 정보 저장 (이름, 담당자, 전화, 주소)
- 저장된 거래처로 명세서 빠르게 생성
- 거래처별 거래 이력 추적

**사용 시나리오:**
```
새로운 거래처 추가
→ "거래처 추가" 버튼 클릭
→ 거래처 정보 입력
→ 등록 완료
→ 다음부터 명세서 생성 시 바로 선택 가능
```

### 4. 발주 (자동 발주)
**기능:**
- 안전재고 미달 품목 자동 감지
- 발주서 자동 생성
- 행정팀에 알림 전송
- 발주 상태 추적 (대기 → 승인 → 완료)

**프로세스:**
```
현황 대시보드 확인
→ 안전재고 미달 품목 표시
→ "자동 발주 생성" 버튼 클릭
→ 발주서 생성
→ 행정팀에 알림 전송
→ 승인 대기
```

### 5. 물품등록 (상세 등록)
**입력 항목:**
- **제품명**: 물품 이름
- **수량**: 초기 입고 수량
- **단가**: 제품 가격
- **거래처**: 공급 업체 선택
- **유효기간**: 의약품/의료용품 만료일
- **LOT번호**: 제조 배치 번호
- **공급내역 보고 대상**: UDI 보고 대상 여부 (체크박스)
- **카테고리**: 물품 분류

**사용 시나리오:**
```
새로운 물품 입고
→ "새 물품 등록" 버튼 클릭
→ 모든 정보 입력
→ 등록 완료
→ 자동으로 재고 추가
```

### 6. 스캔 입고

#### 6-1. 바코드 스캔
**프로세스:**
```
1. "📱 바코드 스캔" 모드 선택
2. "카메라 시작" 버튼 클릭
3. 제품 바코드에 카메라 대기
4. "촬영" 버튼 클릭
5. OCR 인식 완료
   ├─ 제품명 ✓
   ├─ 바코드 ✓
   ├─ 수량 ✓
   ├─ 단가 ✓
   ├─ 유효기간 (추가 입력 필요)
   └─ LOT번호 (추가 입력 필요)
6. 추가 정보 입력 여부 확인
   ├─ "수정하기" → 필요한 정보 입력
   └─ "그대로 진행" → 바로 입고
7. "입고 처리" 버튼 클릭
8. 입고 완료
```

**인식 정보:**
- ✅ 자동 인식: 제품명, 바코드, 수량, 단가
- ⚠️ 수동 입력: 유효기간, LOT번호

#### 6-2. 명세서 스캔
**프로세스:**
```
1. "📄 명세서 스캔" 모드 선택
2. "카메라 시작" 버튼 클릭
3. 명세서 전체를 카메라에 대기
4. "촬영" 버튼 클릭
5. OCR 인식 완료
   ├─ 거래처명 ✓
   ├─ 송장일 ✓
   └─ 품목 목록 (제품명, 수량, 단가) ✓
6. 여러 품목 일괄 표시
7. 추가 정보 필요 여부 확인
8. "입고 처리" 버튼 클릭
9. 모든 품목 일괄 입고 완료
```

**인식 정보:**
- ✅ 자동 인식: 거래처명, 송장일, 품목명, 수량, 단가
- ⚠️ 수동 입력: 유효기간, LOT번호 (필요 시)

#### 6-3. 파일 업로드
카메라가 없는 환경에서는 파일 업로드로 스캔 가능:
```
1. "📁 파일 업로드" 클릭
2. 명세서 또는 바코드 사진 선택
3. OCR 처리
4. 입고 진행
```

## 🔧 Supabase 설정 필수 사항

다음 SQL을 Supabase SQL Editor에서 실행해야 합니다:

```sql
-- SUPABASE_INVENTORY_ADVANCED.md 참고
```

**필수 테이블:**
- `suppliers` - 거래처 정보
- `inventory` (확장) - 유효기간, LOT번호, UDI 대상 여부 등
- `purchase_orders` - 발주서
- `inventory_receipts` - 입고 이력
- `udi_reports` - UDI 보고서

## 📊 워크플로우 예시

### 시나리오 1: 신규 물품 입고
```
1. 물품등록 탭 → "새 물품 등록"
2. 제품명, 수량, 단가, 거래처, 유효기간, LOT번호 입력
3. UDI 대상 여부 체크
4. 등록 완료
5. 재고에 자동 추가
```

### 시나리오 2: 명세서 스캔으로 일괄 입고
```
1. 스캔 입고 탭 → "📄 명세서 스캔" 선택
2. 카메라로 명세서 촬영
3. OCR 인식 (거래처, 품목, 수량, 단가)
4. 추가 정보 필요 여부 확인
5. "입고 처리" 클릭
6. 모든 품목 자동 입고
```

### 시나리오 3: 안전재고 미달 시 자동 발주
```
1. 현황 대시보드 확인
2. "안전재고 미달" 항목 표시
3. "자동 발주 생성" 버튼 클릭
4. 발주서 자동 생성
5. 행정팀에 알림 전송
6. 발주 현황에서 추적
```

### 시나리오 4: UDI 보고
```
1. UDI 보고 탭 클릭
2. UDI 대상 품목 자동 필터링
3. 제품명, 수량, LOT번호, 유효기간 표시
4. "보고서 생성" 버튼 클릭
5. 보고서 완성
```

## ⚠️ 주의사항

1. **유효기간 관리**: 의약품/의료용품은 반드시 유효기간 입력
2. **LOT번호**: 추적 가능성을 위해 입력 권장
3. **UDI 대상 표시**: 의료기기는 반드시 체크
4. **안전재고 설정**: 물품 등록 시 자동 설정 (현재 수량의 30%)
5. **거래처 정보**: 정확한 거래처 정보로 명세서 생성

## 🆘 문제 해결

| 문제 | 해결 방법 |
|:---|:---|
| 카메라 인식 안 됨 | 브라우저 카메라 권한 확인 |
| OCR 인식 실패 | 명세서/바코드 선명도 확인 |
| 제품 입고 안 됨 | 물품등록 탭에서 먼저 등록 |
| 발주서 생성 안 됨 | 안전재고 설정 확인 |
