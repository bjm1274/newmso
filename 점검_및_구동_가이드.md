# SY INC. MSO 통합 시스템 - 점검 및 구동 가이드

## ✅ 점검 완료 사항

### 1. 필수 파일 보완
- **`lib/supabase.ts`**  
  - 프로젝트 전반에서 `@/lib/supabase`를 사용하는데 파일이 없어 **새로 생성**했습니다.  
  - Supabase URL·Anon Key는 환경 변수에서 읽습니다.

### 2. 로그인
- 사번(아이디) + 비밀번호만 사용하도록 수정됨.
- 로그인 성공 시 `setLoading(false)` 후 이동하도록 처리해 두었습니다.

### 3. 메뉴·권한
- **관리자** 메뉴: MSO만 노출 (`company === 'SY INC.'` 또는 `permissions.mso === true`).
- **인사관리** 메뉴: MSO 또는 `permissions.hr === true`인 경우만 노출.
- 관리자 페이지·인사관리 페이지 진입 시 위 권한으로 접근 차단.

### 4. 데이터 로드
- 메인 페이지: `user`, `selectedCompanyId` 변경 시 `fetchERPData` 호출.
- 직원·게시판: `company_id`(또는 비MSO 시 `user.company_id`)로 필터링.
- 전자결재: `company_id` / `selectedCompanyId` 기준 필터 및 상신 시 `company_id` 저장.

### 5. 린트
- `lib`, `main/page.tsx`, `조직도본문`, `login/page.tsx` 등 확인했을 때 **린트 에러 없음**.

## ⚠️ 구동 전 필수 설정

### 1. 환경 변수 (Supabase)
프로젝트 루트에 **`.env.local`** 파일을 만들고 아래를 넣으세요.

```env
NEXT_PUBLIC_SUPABASE_URL=https://YOUR_PROJECT_REF.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your_anon_key_here
```

- Supabase 대시보드 → **Project Settings → API** 에서 URL과 `anon` key를 복사해 사용하면 됩니다.

### 2. DB 마이그레이션
**Supabase SQL Editor**에서 아래 파일을 **순서대로** 실행하세요.

1. **`MSO_통합_DB_스키마_최종.sql`** (또는 기존에 쓰던 최종 스키마)  
   - `staff_members`, `approvals`, `board_posts` 등 기본 테이블이 있어야 합니다.
2. **`MSO_멀티회사_스키마_마이그레이션.sql`**  
   - `companies`, `company_settings`, `leave_policies`, `annual_leave_balances`, `leave_promotion_events`, `payroll`  
   - `staff_members`에 `company_id`, `permissions`, `password` 컬럼 추가  
   - 초기 회사 데이터 및 `staff_members.company_id` 백필

마이그레이션을 하지 않으면 다음이 동작하지 않을 수 있습니다.

- 회사 선택·회사 관리
- 직원별 권한·비밀번호 설정
- 전자결재·직원 목록의 회사별 필터

### 3. 직원 비밀번호
- `staff_members`에 `password` 컬럼이 있어야 로그인 가능합니다.
- 비밀번호가 비어 있으면 "비밀번호가 설정되지 않았습니다" 메시지가 나옵니다.
- **관리자 → 권한 설정**에서 직원 선택 후 **비밀번호 설정**으로 초기 비밀번호를 넣어 주세요.

### 4. MSO 테스트 계정
- 사번 `100`, 비밀번호 `syinc!!` 로 로그인하면 MSO 관리자(하드코딩)로 들어갑니다.
- DB에 MSO 소속 직원을 두고 `permissions.mso = true` 로 설정해도 동일한 권한으로 동작합니다.

## 🔍 런타임에서 확인할 것

1. **로그인**  
   - 사번 + 비밀번호로만 로그인되는지, 잘못된 비밀번호/미설정 비밀번호 시 메시지가 나오는지.
2. **메뉴**  
   - MSO가 아닌 계정으로는 **관리자** 메뉴가 안 보이는지.  
   - 인사관리 권한이 없는 계정에는 **인사관리** 메뉴가 안 보이는지.
3. **회사 선택**  
   - MSO 로그인 시 상단에 회사 선택이 보이고, 선택 시 직원·게시판·전자결재가 해당 회사만 보이는지.
4. **관리자 페이지**  
   - 회사 관리, 권한 설정(인사/MSO/비밀번호 등)이 정상 동작하는지.

## 📌 알려진 제한

- **`package.json`** 이 이 폴더에 없으면 `npm run build` / `npm run dev` 는 상위(또는 실제 Next 프로젝트 루트)에서 실행해야 합니다.
- **`board_posts.company_id`**  
  - 마이그레이션에서 추가되므로, 마이그레이션 전에는 회사별 게시판 필터가 적용되지 않을 수 있습니다.
- **`approvals.company_id`**  
  - 마이그레이션 후 전자결재 목록·상신이 회사별로 필터/저장됩니다.

위 설정과 마이그레이션을 적용한 뒤, 로그인 → 메뉴 → 회사 선택 → 관리자 기능 순으로 한 번씩 실행해 보시면 됩니다.
