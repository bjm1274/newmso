# 병원 ERP 재고관리 시스템 - 완성본

## 🎉 개요

병원 ERP 시스템의 재고관리 기능이 완전히 재구성되었습니다. 모든 버튼과 기능이 정상적으로 작동하며, 요구하신 모든 기능이 구현되었습니다.

---

## ✨ 주요 기능

### 1. **UDI (의료기기 공급내역 보고)**
- 의료기기법에 따른 공급내역 보고 대상 품목 자동 구분
- UDI 대상 품목만 필터링하여 관리
- CSV 형식 보고서 자동 생성 및 다운로드
- LOT번호, 유효기간 자동 포함

### 2. **명세서 자동 작성**
- 거래처 정보 사전 등록 및 저장
- 명세서 작성 시 거래처 정보 원클릭 불러오기
- 품목 자동완성 (재고 목록에서 검색)
- 단가 자동 입력 및 금액 자동 계산
- 인쇄 기능 지원

### 3. **자동 발주**
- 최소 재고 수준 이하로 떨어진 품목 자동 감지
- 발주서 자동 생성 (공급업체별 그룹화)
- 행정팀에 자동 알림 전송
- 발주 리스트 자동 추가
- 발주서 승인/취소 관리

### 4. **스캔 입고**
- **바코드 스캔**: 제품 정보 자동 인식
- **명세서 스캔**: 품목명, 수량, 단가 자동 인식
- ⭐ **명세서 스캔 시 유효기간/LOT번호 추가 입력 여부 확인 창**
  - "추가 입력" 선택 시 품목별로 유효기간/LOT번호 입력 가능
  - "그냥 등록" 선택 시 추가 정보 없이 바로 입고
- 일괄 입고 처리

### 5. **촬영 입고**
- 제품 바코드, 제품명, 유효기간, LOT번호 자동 인식
- OCR 기반 텍스트 인식
- 인식 정보 확인 및 수정 기능
- 자동 입고 처리

### 6. **물품 등록**
- 상세 정보 입력
  - 제품명, 분류, 제조사, 바코드
  - 현재 수량, 최소 재고, 단가
  - 공급업체, LOT번호, 유효기간
  - ⭐ **공급내역 보고 대상 (UDI) 여부 선택**
  - 보관 위치, 비고
- 최소 재고 자동 계산 (현재 수량의 30%)

### 7. **재고 현황**
- 실시간 재고 현황 조회
- UDI 대상 품목 표시
- 안전재고 미달 품목 강조
- 입출고 간편 처리
- 자동 결재 상신 제안

## 📖 사용 방법

### 기본 사용 흐름

1. **물품 등록** 탭에서 제품 등록
   - 제품명, 분류, 공급업체 입력
   - UDI 대상 여부 선택
   - 최소 재고 설정

2. **스캔/촬영** 탭에서 입고 처리
   - 명세서 스캔 → 추가 정보 입력 여부 선택
   - 촬영 → 자동 인식 → 확인 → 입고

3. **재고 현황** 확인
   - 안전재고 미달 품목 확인
   - 필요 시 결재 상신

4. **발주** 탭에서 자동 발주
   - 안전재고 미달 품목 자동 발주서 생성
   - 행정팀에 알림 전송

5. **UDI** 탭에서 공급내역 보고
   - UDI 대상 품목 선택
   - 보고서 생성 및 다운로드

**자세한 사용 방법은 `재고관리_시스템_완성_가이드.md` 파일을 참고하세요.**

## 📊 데이터베이스 구조

### 주요 테이블

| 테이블명 | 설명 |
|---------|------|
| `inventory` | 재고 품목 정보 (확장됨) |
| `suppliers` | 거래처 정보 |
| `purchase_orders` | 발주서 |
| `inventory_receipts` | 입고 이력 |
| `udi_reports` | UDI 보고서 |
| `inventory_logs` | 재고 입출고 로그 |
| `notifications` | 알림 (확장됨) |

### 주요 컬럼 (inventory 테이블)

- `is_udi_reportable`: 공급내역 보고 대상 여부
- `min_stock`: 최소 재고 (안전재고)
- `supplier_id`: 공급업체 ID
- `expiry_date`: 유효기간
- `lot_number`: LOT 번호
- `category`: 분류 (의료기기, 소모품 등)
