## 현재 구조 평가

이미 통합이 잘 된 영역은 유지하는 편이 맞다.

- `인사변동`은 `인사발령 + 포상/징계`를 하나의 스위트로 묶고 있어 구조가 좋다.
  - 근거: `app/main/기능부품/인사관리.tsx:133`, `app/main/기능부품/인사관리.tsx:769`
- `근태`는 분석형 기능을 한 화면 안에서 전환하는 구조라 추가 분해보다 현재 유지가 낫다.
  - 근거: `app/main/기능부품/인사관리.tsx:128`, `app/main/기능부품/인사관리.tsx:802`
- `교대근무`는 운영 탭만 남겨둔 상태라 인사 담당자 입장에서 과하지 않고 적절하다.
  - 근거: `app/main/기능부품/인사관리.tsx:156`, `app/main/기능부품/인사관리.tsx:828`
- `급여`, `계약`은 이미 내부 탭형 워크스페이스라 별도 메뉴로 다시 쪼갤 이유가 크지 않다.
  - 근거: `app/main/기능부품/인사관리.tsx:138`, `app/main/기능부품/인사관리.tsx:144`, `app/main/기능부품/인사관리.tsx:877`, `app/main/기능부품/인사관리.tsx:912`

## 통합 후보

### 1. 문서센터

- 대상:
  - `문서보관함`
  - `증명서`
  - `서류제출`
- 근거:
  - 현재 세 기능이 모두 문서 입수, 문서 보관, 문서 발급 흐름에 속하지만 메뉴가 분산돼 있다.
  - 인사 담당자는 실제로 "직원 문서를 관리한다"는 하나의 업무로 인식할 가능성이 높다.
  - 참고 위치: `app/main/기능부품/인사관리.tsx:935`, `app/main/기능부품/인사관리.tsx:939`, `app/main/기능부품/인사관리.tsx:945`
- 권장 방식:
  - 최상위 메뉴를 `문서센터`로 바꾸고 내부 탭으로 `보관함 / 증명서 / 제출함`을 둔다.
  - `계약`은 법적 계약 흐름이 강하므로 이 센터에 완전 흡수하지 말고 교차 링크만 제공한다.
- 우선순위: 높음

### 2. 자격·안전센터

- 대상:
  - `건강검진`
  - `면허/자격증`
  - `의료기기점검`
  - `사고보고서`
- 근거:
  - 네 기능 모두 점검기한, 증빙, 리스크, 감사 대응 성격이 강하다.
  - 현재는 복무/복지 그룹 아래 나뉘어 있지만 운영 목적은 복지보다 규정 준수에 가깝다.
  - 참고 위치: `app/main/기능부품/인사관리.tsx:905`, `app/main/기능부품/인사관리.tsx:907`, `app/main/기능부품/인사관리.tsx:908`, `app/main/기능부품/인사관리.tsx:910`
- 권장 방식:
- 메뉴명을 `자격·안전센터`로 두고 내부 탭으로 합친다.
  - 상단에 공통 필터를 두고 `직원`, `부서`, `만료예정`, `미조치` 기준으로 한 번에 보는 구조가 좋다.
- 우선순위: 높음

### 3. 입퇴사·교육센터

- 대상:
  - `교육`
  - `오프보딩`
  - 필요 시 `구성원` 일부 요약 카드
- 근거:
  - 교육 이수, 온보딩/오프보딩 체크리스트, 재직 상태 변화는 직원 라이프사이클 관점에서 이어진다.
  - 현재는 `교육`과 `오프보딩`이 독립 메뉴라 업무 흐름이 끊긴다.
  - 참고 위치: `app/main/기능부품/인사관리.tsx:790`, `app/main/기능부품/인사관리.tsx:796`
- 권장 방식:
  - 당장 `구성원`까지 크게 합치기보다 `교육 + 오프보딩`부터 묶고, 직원 상세에서 진입하는 구조로 시작한다.
- 메뉴명 예시: `입퇴사·교육센터`
- 우선순위: 중간

## 통합 비권장 영역

- `급여`
  - 이미 내부 탭이 있고 정책성 기능은 관리자 전용으로 분리된 상태라 추가 통합 효과가 작다.
- `계약`
  - 문서 영역과 관련은 있지만 법적 리스크와 템플릿 흐름이 강해서 별도 워크스페이스를 유지하는 편이 안전하다.
- `캘린더`
  - 인사 전용이 아니라 공유 일정 성격이 강하므로 문서센터에 합치지 않는 편이 낫다.
- `경조사`
  - 현재 단독 메뉴여도 인지 비용이 높지 않고, 다른 복지 기능이 더 생기기 전까지 억지 통합 이점이 작다.

## 추천 실행 순서

1. `문서보관함 + 증명서 + 서류제출`을 `문서센터`로 통합
2. `건강검진 + 면허/자격증 + 의료기기점검 + 사고보고서`를 `자격·안전센터`로 통합
3. `교육 + 오프보딩`을 `입퇴사·교육센터`로 통합

## 결론

- 이번에 `비품대여`를 재고관리로 빼면서 인사관리의 메뉴 역할은 한층 선명해졌다.
- 다음 통합은 `문서`, `규정준수`, `인력생애주기` 3축으로 정리하는 것이 가장 효과적이다.
- 반대로 `급여`, `계약`, `캘린더`는 지금 구조를 유지하는 편이 운영 안정성과 인지 분리 측면에서 더 낫다.
