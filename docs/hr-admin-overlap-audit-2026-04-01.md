# 인사관리 기능 이동 시 관리자 메뉴 중복 점검

작성일: 2026-04-01

## 결론

이동 후보를 관리자 메뉴와 대조한 결과, 바로 중복으로 판단되는 항목은 3개입니다.

- 연차사용촉진 자동화
- 근태 차감 규칙
- 계약 템플릿/계약 설정 일부

나머지 고위험 기능은 관리자 메뉴 안에 아직 동일 기능이 없어서, 기존 화면에 억지로 겹치게 넣기보다 새 관리자 하위 메뉴를 만들거나 기존 관리자 화면을 확장하는 쪽이 맞습니다.

## 실제 중복으로 확인된 항목

### 1. 연차사용촉진 자동화

인사관리 쪽:

- `app/main/기능부품/인사관리서브/휴가신청/연차촉진시스템.tsx:110`
  - `notifications`에 연차촉진 알림을 넣습니다.

관리자 쪽:

- `app/main/기능부품/관리자전용서브/알림자동화설정.tsx:39-90`
  - 연차촉진 1차/2차 알림을 계산해 `notifications`와 `annual_leave_promotion_logs`에 저장합니다.

판단:

- 같은 목적의 기능이 이미 관리자 메뉴에 있습니다.
- 인사관리 화면의 `연차사용촉진 자동화`는 제거하고, 관리자 `알림자동화설정`으로 일원화하는 것이 맞습니다.

### 2. 근태 차감 규칙

인사관리 쪽:

- `app/main/기능부품/인사관리서브/인사통합설정.tsx:136-163`
  - 지각/조퇴 차감, 포괄임금, 보상휴무 성격의 정책을 저장합니다.

관리자 쪽:

- `app/main/기능부품/관리자전용서브/근태차감규칙설정.tsx:11-42`
  - `attendance_deduction_rules`를 조회/저장합니다.
- `app/main/기능부품/관리자전용서브/회사관리.tsx:181-184`
  - 이미 `근태 규칙` 탭으로 노출되어 있습니다.
- `app/main/기능부품/관리자전용서브/회사관리.tsx:354`
  - `AttendanceDeductionRules`를 바로 렌더링합니다.

판단:

- “지각/조퇴 차감 규칙”은 이미 관리자 메뉴에 있습니다.
- 이 부분을 인사통합설정에 남기면 관리자와 인사 화면에 정책 저장入口가 2개 생깁니다.
- 차감 규칙은 관리자 `회사관리 > 근태 규칙`으로 고정하고, 인사통합설정에서는 제거하거나 관리자 링크만 두는 게 맞습니다.

주의:

- `인사통합설정` 전체가 다 중복은 아닙니다.
- `포괄임금 자동화`, `휴일 보상휴무` 같은 나머지 정책은 관리자 쪽 동일 화면이 아직 없습니다.

### 3. 계약 설정 일부

인사관리 쪽:

- `app/main/기능부품/인사관리서브/계약관리.tsx:227-348`
  - 비과세 항목 설정, 연봉 계약 갱신 설정 블록이 들어 있습니다.

관리자 쪽:

- `app/main/기능부품/관리자전용서브/계약관리도구.tsx:86-99`
  - `contract_templates`를 저장합니다.
- `app/main/기능부품/관리자전용서브/회사관리.tsx:193-197`
  - 이미 `계약 설정` 탭이 있습니다.
- `app/main/기능부품/관리자전용서브/회사관리.tsx:358`
  - `ContractManager`를 렌더링합니다.

판단:

- 계약 “표준양식/기본 설정” 성격은 이미 관리자 쪽에 있습니다.
- 다만 인사관리 `계약관리`의 비과세 항목/연봉 갱신 기본값은 계약 템플릿을 넘어 급여정책 성격도 섞여 있어 완전 중복은 아닙니다.
- 그래서 계약 표준양식은 관리자 쪽에 흡수하고, 급여성 설정은 별도 관리자 정책 화면으로 쪼개는 것이 안전합니다.

## 비슷해 보이지만 실제 중복은 아닌 항목

### 1. 연차 자동부여 설정

- 관리자 메뉴에는 `연차수동부여`만 있습니다.
- `app/main/기능부품/관리자전용서브/연차수동부여.tsx:123`
  - 직원별 총량과 사용량을 직접 조정하는 기능입니다.

판단:

- `연차 자동부여 설정`은 규칙 기반 일괄 부여이고, `연차수동부여`는 수동 보정입니다.
- 목적이 달라서 중복이 아닙니다.

### 2. 공휴일 달력 / 휴일·대체휴무 규칙

- 관리자 메뉴에서 이를 직접 다루는 화면은 아직 없습니다.
- `회사관리`에 넣기 좋은 성격이지만 현재 구현은 없습니다.

판단:

- 옮길 수는 있지만, 현재 관리자 메뉴 안에 같은 기능이 있는 것은 아닙니다.

### 3. 급여 정책 기능 전반

- `급여명세서서식관리.tsx`는 문서 서식 관리입니다.
- `급여이상치감지.tsx`는 감지/리포트 화면입니다.
- 관리자 메뉴 어디에도 세율·보험요율, 급여월 잠금, 급여정책버전, 가압류/상계, 프리랜서 지급을 직접 저장하는 기능은 없습니다.

판단:

- 급여 쪽은 중복보다는 “관리자 전용 메뉴 부재”가 문제입니다.
- 새 `급여정책센터`를 만드는 쪽이 맞습니다.

### 4. 근무표 규칙/패턴 저장

- `팀관리.tsx`는 팀 CRUD만 합니다.
- `근무표자동편성`의 규칙 저장/패턴 저장/월간 덮어쓰기를 대체하는 관리자 기능은 없습니다.

판단:

- 이름상 `팀관리`와 가까워 보이지만 실제 중복은 아닙니다.
- 별도 “근무표규칙관리”가 필요합니다.

### 5. 문서보관함 관리

- `시스템마스터센터`에는 위키 버전/운영 통계만 있고, `document_repository`를 직접 수정하는 화면은 없습니다.

판단:

- 문서보관함 생성/수정/삭제는 관리자 쪽에 아직 동일 기능이 없습니다.

## 최종 권고

### 기존 관리자 기능에 흡수할 것

- 연차사용촉진 자동화 → `알림자동화설정`
- 지각/조퇴 차감 규칙 → `회사관리 > 근태 규칙`
- 계약 표준양식/기본 설정 → `회사관리 > 계약 설정`

### 새 관리자 메뉴가 필요한 것

- 연차 자동부여 설정
- 공휴일 달력
- 휴일/대체휴무 규칙
- 인사통합설정의 나머지 정책
- 급여정책센터
- 근무표규칙관리
- 문서정책관리

### 구현 시 주의

- 이미 관리자에 있는 기능은 “복사 이동”이 아니라 기존 관리자 화면으로 통합해야 합니다.
- 그렇지 않으면 같은 데이터를 저장하는 화면이 두 군데가 되어 운영자가 더 헷갈립니다.
- 특히 `연차촉진`, `근태 차감`, `계약 설정`은 중복 메뉴를 만들지 않는 것이 중요합니다.
