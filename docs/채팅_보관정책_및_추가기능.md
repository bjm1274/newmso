# 채팅 보관정책 및 카카오워크 스타일 추가 기능 제안

## 1. 적용된 보관정책

| 구분 | 보관기간 | 비고 |
|------|----------|------|
| **대화기록(텍스트)** | **5년** | 파일 없는 메시지 |
| **사진** | **1년** | file_kind = 'image' |
| **10MB 이하 파일** | **1년** | file_kind = 'file', file_size_bytes ≤ 10MB |
| **동영상 / 10MB 초과 파일** | **3개월** | file_kind = 'video' 또는 size > 10MB |
| **100MB 초과 파일** | **업로드 불가** | 클라이언트에서 차단 |

- DB: `messages` 테이블에 `file_size_bytes`, `file_kind` 컬럼 추가
- 정리 함수: `cleanup_chat_messages_by_retention()` (Supabase SQL)
- API: `GET /api/cron/chat-retention` (Vercel Cron 또는 수동 호출)

---

## 2. Supabase 마이그레이션

1. **컬럼 추가**  
   `supabase_migrations/chat_retention_and_file_meta.sql` 내용을  
   Supabase 대시보드 → SQL Editor에서 실행합니다.

2. **기존 DB만** 컬럼이 없다면:
   ```sql
   ALTER TABLE messages ADD COLUMN IF NOT EXISTS file_size_bytes BIGINT;
   ALTER TABLE messages ADD COLUMN IF NOT EXISTS file_kind VARCHAR(20);
   ```
   그 다음 동일 파일에 있는 `cleanup_chat_messages_by_retention()` 함수 생성 부분도 실행합니다.

## 3. 보관정책 자동 실행 (Cron)

### Vercel Cron

프로젝트 루트에 `vercel.json`이 있다면 예시는 아래와 같습니다.

```json
{
  "crons": [
    {
      "url": "https://erp.pchos.kr/api/cron/chat-retention",
      "schedule": "0 2 * * *"
    }
  ]
}
```

- `0 2 * * *`: 매일 새벽 2시(UTC). 한국시간 새벽 2시는 `0 17 * * *` (전날 17:00 UTC).
- 환경변수에 `CRON_SECRET`을 설정한 경우, 요청 헤더에 `Authorization: Bearer <CRON_SECRET>` 포함해야 합니다.

### 수동 호출

```bash
curl -H "Authorization: Bearer YOUR_CRON_SECRET" https://erp.pchos.kr/api/cron/chat-retention
```

## 5. 카카오워크 스타일 추가 기능 제안

이미 있는 기능: 그룹방, 1:1, 공지방, 파일/이미지 공유, 답글, 이모지 반응, 투표, 고정 메시지, 대화상대 추가, 나가기, 실시간·푸시 알림.

아래는 **추가 검토 시 유용한 기능** 예시입니다.

| 기능 | 설명 | 난이도 |
|------|------|--------|
| **읽음 표시** | 메시지별 읽은 사람/시간 표시 | 중 (message_reads 활용 확대) |
| **메시지 검색** | 방/전체 메시지 키워드 검색 | 중 |
| **알림 방해 금지 시간** | 지정 시간대 알림 무음 | 하 |
| **예약 발송** | 지정 시각에 메시지 전송 | 중 |
| **멘션 알림 강화** | @멘션 시 푸시 + 배지 | 현재 일부 구현됨 |
| **파일 다운로드 기한** | 보관기간과 연동한 다운로드 링크 만료 | 중 |
| **방별 알림 on/off** | 방마다 알림 켜기/끄기 | 현재 room_notification_settings 존재 |
| **메시지 인용(멀티)** | 여러 메시지 동시 인용 | 중 |
| **이모지 확장** | 커스텀 이모지/스티커 | 중 |
| **화상/음성 통화** | WebRTC 등 연동 | 상 |

원하는 항목이 있으면 우선순위만 정해 주시면, 그에 맞춰 설계·구현 단계도 정리해 드리겠습니다.
