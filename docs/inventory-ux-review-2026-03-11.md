# 재고관리 통합 검토 메모 (2026-03-11)

## 중복 또는 겹치는 흐름

- `현황`과 `발주`, `수요예측`
  - 모두 부족 재고를 발견하고 발주로 연결하는 흐름을 갖고 있었습니다.
  - 이번 수정에서 자동기안 로직을 공통화해 같은 기준으로 요청이 올라가도록 맞췄습니다.

- `등록`, `엑셀 일괄 등록`, `AI 명세서 추출`
  - 모두 신규 품목을 시스템에 넣는 흐름인데 진입점이 분산돼 있었습니다.
  - 재고 메인 화면에서 `등록 허브`로 묶어 전환하도록 UI를 재정리했습니다.

- `명세서`, `납품확인서`, `UDI`
  - 모두 조달 문서 흐름인데 서로 다른 그룹에 흩어져 있었습니다.
  - 이번 개편에서 `조달` 그룹으로 모아 같은 맥락에서 접근하게 바꿨습니다.

- `거래처`, `카테고리`, `자산`, `AS/반품`, `소모품 통계`
  - 업무 기준정보와 후속 관리 화면이 분리돼 있어 찾기 어려웠습니다.
  - `기준정보` 그룹으로 재배치했습니다.

## 이번 수정에서 반영한 내용

- 재고관리 상단 UX를 `관제 / 실행 / 조달 / 기준정보` 구조로 재편
- 자동 발주 기안 로직 공통화
- 비MSO 사용자가 전체 재고와 전체 로그를 보던 범위 문제 수정
- 수요예측이 `출고` 로그를 제대로 잡지 못하던 문제 수정

## 추가로 정리하면 좋은 후보

- `재고실사`와 `이관`에서 사용하는 수량 검증 로직도 공통 유틸로 추가 통합
- `거래처`와 `발주` 사이에 공급사별 구매 이력 요약 카드 연결
- `소모품 통계`와 `수요예측`을 하나의 분석 화면으로 재결합 검토
