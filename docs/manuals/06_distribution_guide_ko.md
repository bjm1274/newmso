# NEWMSO 설명서 배포 가이드

## 1. 목적

이 문서는 역할별 설명서를 누구에게 어떤 방식으로 배포하면 되는지 정리한 실행 가이드입니다.

## 2. 배포 파일 구성

| 파일 | 대상 | 용도 |
| --- | --- | --- |
| `00_system_overview_ko.md` | 교육 담당자, 관리자, 마스터 | 전체 기준 문서 |
| `01_employee_manual_ko.md` | 일반 사원 | 실사용 안내 |
| `02_manager_manual_ko.md` | 팀장, 실장, 부장, 간호과장 등 | 승인/운영 안내 |
| `03_admin_manual_ko.md` | 관리자 | 운영 실무 안내 |
| `04_master_manual_ko.md` | 마스터 | 최고권한 운영 기준 |
| `05_distribution_messages_ko.md` | 배포 담당자 | 공지문 복사본 |

## 3. 권장 배포 순서

### 1단계

관리자와 마스터가 먼저 문서를 검토합니다.

- 관리자: `03_admin_manual_ko.md`
- 마스터: `04_master_manual_ko.md`
- 함께 확인할 기준 문서: `00_system_overview_ko.md`

목적:

- 실제 운영 정책과 문서 표현이 맞는지 확인
- 민감 기능 설명 문구 조정
- 내부 용어 통일

### 2단계

부서장에게 먼저 배포합니다.

- 배포 문서: `02_manager_manual_ko.md`
- 공지문 초안: `05_distribution_messages_ko.md`의 부서장 공지문

목적:

- 사원 문의가 들어오기 전에 부서장이 먼저 구조를 이해하도록 준비

### 3단계

일반 사원에게 배포합니다.

- 배포 문서: `01_employee_manual_ko.md`
- 공지문 초안: `05_distribution_messages_ko.md`의 일반 사원 공지문

목적:

- 실제 사용 기능만 중심으로 안내
- 권한 문의와 오류 문의 경로를 먼저 알려 혼선을 줄임

## 4. 역할별 첨부 권장안

### 일반 사원

- 필수 첨부: `01_employee_manual_ko.md`
- 선택 첨부: 없음

### 부서장

- 필수 첨부: `02_manager_manual_ko.md`
- 선택 첨부: `00_system_overview_ko.md`

### 관리자

- 필수 첨부: `03_admin_manual_ko.md`
- 선택 첨부: `00_system_overview_ko.md`

### 마스터

- 필수 첨부: `04_master_manual_ko.md`
- 선택 첨부: `00_system_overview_ko.md`

## 5. 배포 채널별 권장 방식

### 메신저/카카오톡

- `05_distribution_messages_ko.md`의 짧은 공지문 사용
- 문서는 파일로 첨부
- 질문은 부서장 또는 관리자에게 모아 달라고 안내

### 이메일

- `05_distribution_messages_ko.md`의 사내 공지형 문구 사용
- 역할별 문서를 첨부
- 배포 날짜와 적용 날짜를 명확히 기재

### 사내 게시판

- 공지사항에 설명서 배포 공지 게시
- 첨부 파일로 역할별 설명서 업로드
- 가장 많이 받을 질문 3~5개를 본문에 같이 넣으면 효과적

## 6. 배포 시 함께 적으면 좋은 안내 문구

- 보이는 메뉴는 권한에 따라 다를 수 있습니다.
- 메뉴가 없다고 기능이 없는 것은 아니며, 권한 미부여일 수 있습니다.
- 급여, 증명서, 인사 정보 오류는 인사 담당자 또는 관리자에게 문의합니다.
- 권한 문의는 관리자에게 요청합니다.
- 관리자 화면과 고위험 기능은 별도 운영 기준을 따릅니다.

## 7. 배포 전 최종 확인 체크리스트

1. 현재 배포할 프로그램 버전이 원본 `newmso`와 맞는지 확인
2. 문서 제목과 대상 역할이 맞는지 확인
3. 관리자/마스터 문서에 민감 정보가 과도하게 적히지 않았는지 확인
4. 문의 대상과 연락 경로가 조직에 맞게 정리됐는지 확인
5. 게시판, 메신저, 이메일 중 실제 배포 채널을 정했는지 확인

## 8. 권장 운영 방식

가장 안정적인 배포 방식은 아래 순서입니다.

1. 관리자/마스터 선검토
2. 부서장 선배포
3. 일반 사원 배포
4. 1주일 동안 문의 수집
5. 자주 묻는 질문만 별도 공지로 추가 배포

이 순서로 가면 사원 문의가 바로 관리자에게 몰리지 않고, 부서장 선에서 1차 정리가 가능합니다.
