# NEWMSO 전체 기능 설명서

## 1. 문서 목적

이 문서는 `C:\Users\baek_\newmso` 원본 프로그램 기준으로 현재 제공 중인 기능을 전체적으로 설명하는 기준 문서입니다.  
사원용, 부서장용, 관리자용, 마스터용 설명서는 이 문서를 바탕으로 역할별로 정리한 문서입니다.

## 2. 현재 메인 구조

현재 원본 프로그램의 기본 메인 메뉴는 아래와 같습니다.

| 메인 메뉴 | 주 용도 |
| --- | --- |
| 내정보 | 개인 업무와 개인 기록 확인 |
| 추가기능 | 보조 업무 화면과 외부 링크 |
| 채팅 | 공지 메시지, 1:1/그룹 채팅 |
| 게시판 | 공지, 자유게시판, 일정형 게시판 |
| 전자결재 | 기안, 승인, 결재 캘린더, 양식 관련 기능 |
| 인사관리 | 인력, 근태, 급여, 복지, 문서 관리 |
| 재고관리 | 재고, 발주, 거래처, 자산, 유통기한 관리 |
| 관리자 | 회사 운영, 권한, 백업, 감사, 팝업, 양식 관리 |

별도로 `조직도` 화면이 존재하지만, 현재 원본 사이드바의 고정 메인 메뉴에는 포함되어 있지 않습니다.  
조직도는 `추가기능`의 조직도 카드, 검색 결과 이동, 직접 진입 흐름으로 열리는 구조에 가깝습니다.

## 3. 권한 구조

현재 코드 기준 권한은 크게 아래 수준으로 이해하면 됩니다.

| 구분 | 설명 |
| --- | --- |
| 기본 사용자 | 내정보 중심 사용 |
| 메뉴 권한 | 채팅, 게시판, 전자결재, 인사관리, 재고관리, 관리자 메뉴 노출 여부 결정 |
| 인사 세부 권한 | 구성원, 근태, 급여, 계약, 증명서 등 인사관리 내부 기능 노출 결정 |
| MSO/최고권한 | 전체 회사 또는 관리자 수준 운영 기능 접근 |

현재 원본에서는 설계상 권한 함수가 존재해도, 일부 메뉴는 실제 UI에서 넓게 보이는 상태가 있습니다.  
대표적으로 마이페이지 일부 탭과 전자결재 관리성 탭은 화면상 나타날 수 있으므로, 안내 문서에서는 현재 보이는 구조를 우선 기준으로 삼는 것이 안전합니다.

## 4. 메인 메뉴별 상세 설명

### 4-1. 내정보

`내정보`는 가장 자주 사용하는 개인 시작 화면입니다.

현재 탭 구성은 아래 순서입니다.

| 탭 | 설명 |
| --- | --- |
| 프로필 | 내 기본 정보와 역할별 대시보드 |
| 출퇴근 | 당일 출퇴근 기록과 정정 연계 |
| 할 일 | 개인 할 일 메모 |
| 증명서 | 증명서 신청 및 이력 확인 |
| 급여 | 급여명세 확인 |
| 서류제출 | 개인 서류 제출 |
| 알림 | 시스템 알림, 승인 알림 확인 |

주요 특징은 아래와 같습니다.

- 역할별 대시보드가 다르게 보입니다.
- 즐겨찾기 메뉴를 등록해 자주 쓰는 화면으로 바로 이동할 수 있습니다.
- 근로계약 대기 건이 있으면 전자서명 모달이 열릴 수 있습니다.
- 출퇴근 정정, 증명서, 서류 제출 등은 다른 업무 메뉴와 연결됩니다.

### 4-2. 조직도

조직도는 별도 전용 화면으로 존재합니다.

주요 기능은 아래와 같습니다.

- 회사별 보기
- 이름, 부서 검색
- 팀 필터
- 피라미드형, 목록형, 캔버스형 보기
- 관리자 편집 모드
- 회사별 조직 구조 불러오기

현재 구조상 사이드바 메인 메뉴로 고정 노출되기보다 빠른 진입 화면에 가깝습니다.

### 4-3. 추가기능

추가기능은 업무 보조용 카드 모음입니다. 현재 원본 기준 카드 구성은 아래와 같습니다.

| 카드 | 설명 | 비고 |
| --- | --- | --- |
| 조직도 | 조직도 화면 바로가기 | 공용 |
| 부서별 재고 | 부서 단위 물품/장비 현황 | 공용 |
| 근무현황 | 현재 근무 상태 확인 | 공용 |
| 인계노트 | 인수인계 공유 메모 | 제한 기능 |
| 퇴원심사 | 퇴원 관련 체크리스트/분석 | 부서별 활용 |
| 마감보고 | 일일 마감 보고 | 제한 기능 |
| 직원평가 | 직원 평가 기록 | 제한 기능 |

추가로 외부 링크가 함께 있습니다.

- `KM Park`
- `U+ 웹팩스`

제한 기능은 부서, 직책, 관리자 여부에 따라 숨겨질 수 있습니다.

### 4-4. 채팅

채팅은 사내 실시간 커뮤니케이션 중심 기능입니다.

주요 기능은 아래와 같습니다.

- 공지메시지 방
- 1:1 채팅
- 그룹 채팅
- 이미지, 동영상, 일반 파일 전송
- 멘션
- 북마크
- 반응 이모지
- 투표 생성과 투표 집계
- 핀 고정
- 메시지 전달
- 스레드형 답변
- 공지 등록
- 전자결재용 내용 복사

추가 특징

- `공지메시지` 방은 현재 `대표`, `부장`, `팀장`, `실장`, `병원장`, `이사`, `본부장`, `총무부장`, `진료부장`, `간호부장`만 작성할 수 있습니다.
- 공지방은 직원이 임의로 나갈 수 없도록 막혀 있습니다.
- 입력창의 슬래시 기능으로 `연차 요청 초안`과 `발주 요청 초안`을 만들 수 있습니다.

### 4-5. 게시판

게시판은 공지와 커뮤니티, 일정형 게시판을 함께 제공합니다.

실제 게시판 내부 종류는 아래 7개입니다.

| 게시판 | 설명 |
| --- | --- |
| 공지사항 | 공식 공지 게시 |
| 자유게시판 | 자유 글 공유 |
| 익명소리함 | 익명 의견 수집 |
| 경조사 | 경조사 소식 |
| 수술일정 | 수술 일정 게시 |
| MRI일정 | MRI 일정 게시 |
| 직원제안함 | 제안 접수 |

주요 특징

- 수술일정과 MRI일정은 일정형 필드가 포함됩니다.
- 수술/MRI 템플릿과 부위 필터를 활용할 수 있습니다.
- 익명소리함은 현재 코드상 관리자나 인사 권한이 아니면 목록을 비워서 보여주는 구조입니다.

### 4-6. 전자결재

전자결재는 결재 작성과 승인 기능을 담당합니다.

현재 원본 UI 기준 기본 뷰는 아래 7개가 모두 존재합니다.

| 탭 | 설명 |
| --- | --- |
| 기안함 | 내가 올린 문서 확인 |
| 결재함 | 내가 처리할 문서 확인 |
| 작성하기 | 새 결재 작성 |
| 결재 캘린더 | 달력 기준 결재 확인 |
| 양식빌더 | 결재 양식 설계 |
| 전자서명 | 서명 관련 안내/기능 |
| 직인관리 | 직인 관리 |

현재 UI에서는 관리성 탭도 함께 노출될 수 있으므로, 실제 배포 안내 시에는 역할별 설명서를 함께 사용하는 것이 좋습니다.

기본 결재 양식은 아래가 포함됩니다.

- 인사명령
- 연차/휴가
- 연차계획서
- 연장근무
- 물품신청
- 수리요청서
- 업무기안
- 업무협조
- 양식신청
- 출결정정

주요 특징

- 작성 중 초안 자동저장
- 마지막 기안 불러오기
- 결재선 템플릿 저장
- 일괄 승인 처리
- 결재 상태 필터
- 양식신청 자동 발급 연계
- 연차 승인 시 근태/연차 데이터 연계

### 4-9. 관리자

관리자 메뉴는 현재 원본 코드상 사실상 MSO 권한 계정 중심으로 동작합니다.  
화면 구조는 `대분류 탭 + 내부 보조 탭` 형태입니다.

#### 경영분석

내부 탭

- 경영대시보드
- 재무대시보드
- 예산관리
- 통합보고서
- 법인손익

#### 감사센터

내부 탭

- 접근감사로그
- 감사로그

| 회사관리 | 회사/조직 설정 |

| 팝업관리 | 외부 팝업 데이터 관리 |
| 데이터백업 | 백업 |
| 데이터초기화 | 고위험 초기화 |
| 양식빌더 | 전자결재 양식 설계 |
| 문서서식 | 급여/문서 서식 관리 |

관리자 메뉴 주의점

- `menu_관리자` 권한 키가 있어도, 현재 관리자 화면 자체는 MSO 권한 기준으로 진입을 막는 구조입니다.
- 데이터초기화는 운영 환경에서 매우 위험한 기능입니다.
- 직원권한, 양식빌더, 문서서식, 팝업관리는 실제 운영 화면에 영향을 크게 줍니다.

| 일반 사원 | 내정보, 채팅, 게시판, 전자결재 |
| 부서장 | 내정보, 전자결재, 인사관리, 재고관리, 추가기능 |
| 관리자 | 관리자, 인사관리, 전자결재, 재고관리 |
| 마스터 | 관리자 전체 + 회사 전반 통제 |

## 6. 현재 원본 구조에서 꼭 알아둘 점

- 조직도는 기능은 존재하지만 현재 메인 사이드바 고정 메뉴는 아닙니다.
- 전자결재 관리 탭은 현재 UI에서 일반 사용자에게도 보일 수 있습니다.
- 마이페이지 탭은 현재 UI에서 비교적 넓게 보일 수 있으며, 실제 데이터 범위는 회사 설정과 권한에 따라 달라질 수 있습니다.
- 인사관리는 권한 기반 노출이 가장 잘 연결된 영역입니다.
- 관리자 화면은 사실상 MSO 운영자용으로 보는 것이 정확합니다.
