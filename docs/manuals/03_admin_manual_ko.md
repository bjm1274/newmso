# NEWMSO 관리자용 기능 설명서

## 1. 문서 목적

이 문서는 현재 원본 NEWMSO의 관리자 운영 기능을 설명하는 문서입니다.  
현재 코드 기준으로 관리자 화면은 사실상 MSO 권한 계정 중심으로 동작하므로, 일반 부서 관리자와 최고권한 운영자를 구분해서 이해하는 것이 중요합니다.

## 2. 관리자 메뉴 전체 구조

현재 관리자 메뉴는 아래 구조로 나뉩니다.

### 경영분석

내부 탭

- 경영대시보드
- 재무대시보드
- 예산관리
- 통합보고서
- 법인손익

### 감사센터

내부 탭

- 접근감사로그
- 감사로그

### 직접 실행 탭

- 엑셀등록
- 알림자동화
- 연차수동부여
- 회사관리
- 직원권한
- 수술검사템플릿
- 팝업관리
- 데이터백업
- 데이터초기화
- 양식빌더
- 문서서식
- 급여이상치
- 공문서대장

## 3. 화면별 설명

### 경영대시보드

- 운영과 인사 성격의 핵심 수치를 확인합니다.
- 전체 현황 요약판으로 사용합니다.

### 재무대시보드

- 재무 흐름과 마감 성격 지표를 봅니다.
- 숫자 이상 여부를 점검할 때 사용합니다.

### 예산관리

- 예산 항목과 기준을 관리합니다.
- 회사 단위 지출 통제와 연결될 수 있습니다.

### 통합보고서

- 여러 운영 지표를 한 번에 확인하는 리포트 화면입니다.

### 법인손익

- 회사 단위 손익 흐름을 보는 화면입니다.

### 접근감사로그

- 누가 어떤 데이터나 화면에 접근했는지 추적합니다.
- 보안 및 권한 감사에 중요합니다.

### 감사로그

- 변경 흔적을 확인하는 로그 화면입니다.
- 설정 변경, 데이터 조정, 운영 이슈 조사에 사용합니다.

### 엑셀등록

- 대량 데이터를 한 번에 올리는 기능입니다.
- 소량 수정용이 아니라 일괄 입력용으로 보는 것이 맞습니다.

### 알림자동화

- 자동 알림 규칙을 관리합니다.
- 반복성 안내를 시스템화할 때 사용합니다.

### 연차수동부여

- 특정 직원의 연차를 수동 보정합니다.
- 자동 계산과 다르게 수동 반영되므로 사유 관리가 필요합니다.

### 회사관리

- 회사 기본 정보, 조직 구조, 운영 관련 설정을 다룹니다.

### 직원권한

- 직원별 역할과 권한을 조정합니다.
- 메뉴 노출과 기능 사용 범위에 직접 영향이 있습니다.

### 수술검사템플릿

- 수술 및 MRI 게시판/운영과 연동되는 템플릿을 관리합니다.
- 의료기관 운영 기능입니다.

### 팝업관리

- 외부 소비 지점에서 쓸 팝업 데이터를 등록합니다.
- 홈페이지 노출과 연결될 수 있으므로 대외 노출 정보로 취급해야 합니다.

### 데이터백업

- 운영 데이터 백업을 수행합니다.
- 구조 변경 전 반드시 확인하는 것이 좋습니다.

### 데이터초기화

- 매우 위험한 기능입니다.
- 테스트 환경이 아니라면 별도 승인 없이 사용하면 안 됩니다.

### 양식빌더

- 전자결재 문서 양식을 설계합니다.
- 결재 문서 구조가 바뀌므로 영향 범위가 큽니다.

### 문서서식

- 급여명세서나 문서 출력 형식을 조정합니다.

### 급여이상치

- 급여 데이터의 이상 징후를 탐지합니다.

### 공문서대장

- 공문서 발송 또는 관리 이력을 운영합니다.

## 4. 관리자가 함께 알아야 할 일반 메뉴

관리자는 관리자 메뉴만 보면 부족합니다. 아래 메뉴도 함께 이해해야 운영이 됩니다.

| 메뉴 | 관리자 관점의 확인 포인트 |
| --- | --- |
| 내정보 | 관리자 대시보드 요약 |
| 전자결재 | 결재 적체, 양식 구조, 승인 흐름 |
| 인사관리 | 인사, 근태, 급여, 증명서, 문서 운영 |
| 재고관리 | 부족 재고, 발주, 자산, 유통기한 |
| 채팅 | 공지메시지 운영 |
| 게시판 | 공식 공지 전달 |

## 5. 관리자 일일 체크리스트

1. 결재 대기 건수 확인
2. 긴급 재고 부족 확인
3. 공지 메시지와 게시판 공지 정합성 확인
4. 사용자 권한 문의 확인
5. 장애 또는 데이터 오류 제보 확인

## 6. 관리자 주간 체크리스트

1. 급여이상치 점검
2. 접근감사로그 검토
3. 감사로그 검토
4. 백업 상태 확인
5. 팝업, 양식, 문서서식 최신 상태 확인

## 7. 고위험 기능 운영 원칙

| 기능 | 위험 요소 | 원칙 |
| --- | --- | --- |
| 직원권한 | 잘못 주면 데이터 노출 가능 | 최소 권한으로 설정 |
| 연차수동부여 | 휴가 데이터 왜곡 가능 | 사유 기록 후 사용 |
| 엑셀등록 | 대량 오류 가능 | 샘플 검증 후 업로드 |
| 데이터초기화 | 운영 데이터 손실 | 운영 환경 사용 금지 수준으로 관리 |
| 양식빌더 | 결재 구조 변경 | 실제 사용자 영향 검토 후 반영 |
| 문서서식 | 출력 문서 변경 | 출력물 검수 후 반영 |
| 팝업관리 | 외부 노출 정보 | 게시 전 문구 검수 |

## 8. 꼭 알아둘 현재 구조상 특이점

- 관리자 메뉴는 권한 키와 별개로 실제 화면에서 `MSO 권한` 기준으로 차단됩니다.
- 현재 원본에서는 전자결재 관리성 탭이 일반 사용자에게도 보일 수 있으므로, 관리자 교육 시 역할 구분 설명이 필요합니다.
- 데이터초기화는 현재 화면에 존재하므로, 운영 조직 내 사용 기준을 별도로 정해 두는 것이 좋습니다.

## 9. 문의 기준

| 상황 | 문의 대상 |
| --- | --- |
| 권한 설계 충돌 | 마스터 운영자 |
| 운영 데이터 오류 | 마스터 운영자 또는 총괄 관리자 |
| 배포 후 이상 | 개발 담당 또는 마스터 운영자 |
| 개인정보 노출 우려 | 즉시 마스터 운영자 보고 |
