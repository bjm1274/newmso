## Issues Found

### Fixed

1. Cron routes were callable without authorization when `CRON_SECRET` was missing.
   - Files:
     - `app/api/cron/backup/route.ts`
     - `app/api/cron/chat-retention/route.ts`
   - Risk:
     - Backup could be triggered publicly.
     - Chat retention cleanup could be triggered publicly.
   - Fix:
     - Routes now return `500` when `CRON_SECRET` is not configured.
     - Authorization is now always enforced when the route is enabled.
   - Regression coverage:
     - `tests/e2e/smoke.desktop.spec.ts`

2. Invoice extraction used an outdated Gemini model and failed with `404 model not found`.
   - File:
     - `app/api/extract-invoice/route.ts`
   - Risk:
     - Invoice extraction endpoint was unavailable even with valid auth and API key.
   - Fix:
     - Switched to a fallback model list:
       - `gemini-2.5-pro`
       - `gemini-2.5-flash`
     - Added retry fallback across supported models.
   - Post-fix result:
     - The endpoint no longer fails with model `404`.
     - With a non-invoice sample image, it now fails at JSON extraction instead of model lookup.

### Remaining Operational Risks

1. `app/api/chart-ocr/route.ts`
   - During live API audit, Gemini Vision returned quota exhaustion errors.
   - User impact:
     - OCR may fail at runtime until Gemini quota is restored.

2. `app/api/extract-invoice/route.ts`
   - The model error is fixed, but end-to-end success still needs a representative invoice or statement sample file.
   - Current audit used a non-invoice image only to verify auth and model availability.
