# SY INC. MSO — 디자인 통일 현황

> `app/globals.css` 기준(토스/메디플로우 스타일)으로 페이지별 통일 적용 여부 정리.  
> 디자인 통일 작업 이어갈 때 이 파일을 검색해 보시면 됩니다.

---

## 1. 디자인 기준 (globals.css)

| 구분 | 내용 |
|------|------|
| **글꼴** | Pretendard 전역 |
| **메인 액션** | `--toss-blue` (#3182F6) |
| **보조/비활성** | `--toss-gray-1` ~ `--toss-gray-4` |
| **위험** | `--toss-danger` |
| **카드** | `border 1px --toss-border`, `rounded-lg`(8px) / `.card-toss`(16px) |
| **버튼** | `.btn-toss-primary`, `.btn-toss-secondary`, `.btn-toss-danger` (rounded 12px) |
| **탭** | `.tab-bar` + `.tab-btn` / `.tab-btn-active` |
| **페이지 제목** | `.page-title`, `.page-header-title`, `.caption` |
| **레이아웃** | `.app-page`, `.app-header`, `.app-tab-bar`, `.app-card` |

**치환 규칙**:  
- `bg-gray-50` → `bg-[var(--toss-gray-1)]` 또는 입력창은 `bg-[var(--input-bg)]`  
- `border-gray-200` → `border-[var(--toss-border)]`  
- `text-gray-500` / `text-gray-600` → `text-[var(--toss-gray-3)]` / `text-[var(--toss-gray-4)]`  
- 메인 버튼 → `btn-toss-primary`, 취소/보조 → `btn-toss-secondary`

## 2. 통일 적용 완료

- **2026-02-23 전체 프로그램 일괄 적용**
  - **app/** 하위 모든 `.tsx`: `border-gray-*`, `bg-gray-*`, `text-gray-*`, `bg-blue-*`, `text-blue-*`, `focus:ring-blue-*`, `divide-gray-100` 등 → `var(--toss-*)` / `var(--foreground)` / `var(--input-bg)` 로 치환 완료.
  - PowerShell 일괄 치환 53개 파일 + 수동 보정(근무형태관리, 명세서디자인, OrgChart, 메신저, 게시판 등) + Node 스크립트(border-gray-400/800/900, divide-gray-100 등) 적용.
- **이미 적용되어 있던 화면**: 인사관리 메인, 급여관리·급여명세 하위, 근무형태관리 등.

## 4. 잔여/참고 (일부만 남은 경우)

- `bg-white`는 **globals.css**에서 다크 모드 시 `var(--toss-card)`로 대응되므로 유지해도 됨.
- 세미antic 색(예: 공제 영역 `red-*`)은 의도적 구분용으로 둔 경우가 있음.
- 추가로 남은 gray/blue가 보이면 위 치환 규칙으로 같은 방식 적용하면 됨.

---

## 5. 이어서 할 때

1. 위 **4번** 목록에서 한 화면씩 선택해  
   - `text-gray-*`, `bg-gray-*`, `border-gray-*` → `var(--toss-*)` 또는 통일 클래스로 치환  
   - 메인/취소 버튼 → `btn-toss-primary` / `btn-toss-secondary`  
   - 카드/모달 → `card-toss`, `bg-[var(--toss-card)]`, `border-[var(--toss-border)]`
2. 적용한 파일을 **2번** 표에 추가해 두면 다음에 이어가기 좋습니다.
