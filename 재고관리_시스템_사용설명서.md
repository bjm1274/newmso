## 기본 관리자 계정 (사번 체계 적용)

시스템 설치 후 즉시 사용 가능한 기본 관리자 계정입니다. 본 시스템은 **'사번' 또는 '이름'**을 아이디로 사용합니다.

### 사번 할당 규칙
- **관리자**: 1 ~ 10번
- **부서장**: 11 ~ 20번
- **일반 직원**: 21 ~ 9999번

### 초기 관리자 정보
- **로그인 아이디**: `1` (또는 `박철홍`)
- **비밀번호**: `qkrcjfghd!!`
- **권한**: 병원장 (전체 관리 권한)

## 시스템 개요

### 🎯 목적
병원의 복잡한 재고 관리를 자동화하고, 의료기기 공급내역 보고(UDI)를 간소화하며, 안전재고 관리를 통해 진료 차질을 방지합니다.

### 🏗️ 시스템 구조

```
재고관리 시스템
│
├─ 📊 현황 대시보드
│  ├─ 전체 품목 수
│  ├─ UDI 대상 품목 수
│  ├─ 안전재고 미달 품목 수
│  └─ 자동 발주 생성
│
├─ 🏥 UDI 보고
│  ├─ 의료기기 공급내역 자동 수집
│  ├─ 보고서 생성
│  └─ 제출 관리
│
├─ 📄 명세서 (거래처 관리)
│  ├─ 거래처 정보 저장
│  ├─ 거래처별 명세서 생성
│  └─ 거래처 이력 관리
│
├─ 📦 발주
│  ├─ 자동 발주서 생성
│  ├─ 발주 상태 추적
│  └─ 행정팀 알림
│
├─ 📝 물품등록
│  ├─ 신규 물품 등록
│  ├─ 상세 정보 입력
│  └─ 유효기간/LOT 관리
│
└─ 📱 스캔 입고
   ├─ 바코드 스캔
   ├─ 명세서 스캔
   └─ 파일 업로드
```

### 👥 사용자별 권한

| 역할 | 조회 | 등록 | 수정 | 삭제 | 승인 |
|:---:|:---:|:---:|:---:|:---:|:---:|
| 일반 직원 | ✅ | ❌ | ❌ | ❌ | ❌ |
| 의료진 | ✅ | ⚠️ | ❌ | ❌ | ❌ |
| 행정팀 | ✅ | ✅ | ✅ | ✅ | ✅ |
| 병원장 | ✅ | ✅ | ✅ | ✅ | ✅ |

> **⚠️ 참고**: 의료진은 스캔 입고만 가능합니다.

## 초기 설정

### 1단계: Supabase 데이터베이스 설정 (이름 기반 로그인)

본 시스템은 별도의 이메일 인증 없이 **데이터베이스에 등록된 이름과 비밀번호**만으로 즉시 로그인이 가능합니다. 

1.  **Supabase SQL Editor**에서 `통합_재고관리_DB_스키마.sql` 파일을 실행합니다.
2.  로그인 화면에서 아이디 칸에 **'박철홍'**, 비밀번호 칸에 **'qkrcjfghd!!'**를 입력합니다.
3.  별도의 계정 생성 절차 없이 바로 관리자 권한으로 접속됩니다.

### 1단계: Supabase 데이터베이스 설정

**필수 작업:**
1. Supabase 프로젝트 접속
2. SQL Editor 열기
3. `SUPABASE_COMPLETE_SQL.md` 파일의 모든 SQL 명령어를 **순서대로** 실행
4. 각 단계 완료 후 오류 확인

**예상 소요 시간**: 10~15분

### 2단계: 거래처 등록

**단계:**
1. 재고관리 시스템 → **📄 명세서** 탭
2. **+ 거래처 추가** 버튼 클릭
3. 다음 정보 입력:
   - 거래처명 (필수)
   - 담당자명
   - 전화번호
   - 주소
4. **거래처 등록** 버튼 클릭

**예시:**
```
거래처명: 의료용품 A사
담당자: 김철수
전화: 02-1234-5678
주소: 서울시 강남구 테헤란로 123
```

### 3단계: 물품 등록

**단계:**
1. 재고관리 시스템 → **📝 물품등록** 탭
2. **+ 새 물품 등록** 버튼 클릭
3. 다음 정보 입력:

| 항목 | 설명 | 필수 | 예시 |
|:---|:---|:---:|:---|
| 제품명 | 물품 이름 | ✅ | 인슐린 주사기 |
| 수량 | 초기 입고 수량 | ✅ | 100 |
| 단가 | 제품 가격 | ✅ | 5000 |
| 거래처 | 공급 업체 | ✅ | 의료용품 A사 |
| 유효기간 | 만료일 (의약품/의료용품) | ⚠️ | 2026-12-31 |
| LOT번호 | 제조 배치 번호 | ⚠️ | LOT-2025-001 |
| 공급내역 보고 대상 | UDI 대상 여부 | ⚠️ | ✓ (의료기기만) |

4. **물품 등록** 버튼 클릭

**팁:**
- 의약품/의료용품은 반드시 유효기간 입력
- 의료기기는 "공급내역 보고 대상" 체크
- LOT번호는 추적 가능성을 위해 입력 권장

## 기본 기능

### 1️⃣ 현황 대시보드

**목적**: 재고 상황을 한눈에 파악

**화면 구성:**
```
┌─────────────────────────────────────────┐
│ 📊 현황 대시보드                          │
├─────────────────────────────────────────┤
│                                         │
│  전체 품목: 45개    UDI 대상: 12개      │
│  안전재고 미달: 3개  거래처: 4개        │
│                                         │
├─────────────────────────────────────────┤
│ ⚠️ 안전재고 미달 품목                    │
│                                         │
│ • 인슐린 주사기 (현재: 40개 / 최소: 50개) │
│ • 수술용 장갑 (현재: 80개 / 최소: 100개)  │
│ • 소독용 알콜 (현재: 30개 / 최소: 50개)   │
│                                         │
│ [자동 발주 생성] 버튼                   │
└─────────────────────────────────────────┘
```

**사용 방법:**
1. 메인 화면에서 **📊 현황** 탭 선택
2. 현재 재고 상황 확인
3. 안전재고 미달 품목 확인
4. 필요시 **자동 발주 생성** 버튼 클릭

**자동 발주 생성 프로세스:**
```
[자동 발주 생성] 클릭
    ↓
안전재고 미달 품목 자동 감지
    ↓
발주서 자동 생성
    ↓
행정팀에 알림 전송
    ↓
발주 현황에서 추적 가능
```

### 2️⃣ UDI 보고

**목적**: 의료기기 공급내역 보고 자동화

**사용 시나리오:**
- 월 1회 또는 필요시 의료기기 공급내역 보고

**단계:**
1. 재고관리 시스템 → **🏥 UDI 보고** 탭
2. UDI 대상 품목 자동 필터링 (공급내역 보고 대상으로 등록된 의료기기)
3. 다음 정보 확인:
   - 제품명
   - 수량
   - LOT번호
   - 유효기간

4. **보고서 생성** 버튼 클릭
5. 생성된 보고서 확인 및 제출

**화면 예시:**
```
┌──────────────────────────────────────────┐
│ 🏥 UDI 보고                               │
├──────────────────────────────────────────┤
│ 의료기기 공급내역 보고                    │
│                                          │
│ 제품명: 인슐린 주사기                     │
│ 수량: 100개                              │
│ LOT번호: LOT-2025-001                    │
│ 유효기간: 2026-12-31                     │
│ ────────────────────────────────────────│
│ 제품명: 혈당 측정기                       │
│ 수량: 30개                               │
│ LOT번호: LOT-2025-005                    │
│ 유효기간: 2027-03-31                     │
│                                          │
│ [보고서 생성]                             │
└──────────────────────────────────────────┘
```

### 3️⃣ 명세서 (거래처 관리)

**목적**: 거래처 정보 저장 및 명세서 빠른 생성

**거래처 추가:**
1. **+ 거래처 추가** 버튼 클릭
2. 거래처 정보 입력
3. **거래처 등록** 완료

**명세서 생성:**
1. 거래처 목록에서 원하는 거래처 선택
2. **명세서 생성** 버튼 클릭
3. 필요한 품목 선택
4. 명세서 생성 완료

---

### 4️⃣ 발주

**목적**: 안전재고 미달 시 자동 발주

**자동 발주 프로세스:**
```
안전재고 미달 감지
    ↓
[자동 발주 생성] 클릭
    ↓
발주서 자동 생성
    ↓
행정팀에 알림 전송
    ↓
발주 현황 탭에서 추적
    ↓
승인 대기 중 → 승인 → 완료
```

**발주 현황 확인:**
1. 재고관리 시스템 → **📦 발주** 탭
2. 발주서 목록 확인
3. 상태별 필터링:
   - 대기: 승인 대기 중
   - 승인: 승인됨
   - 완료: 입고 완료

**발주서 상세 정보:**
```
발주 #ABC123
거래처: 의료용품 A사
상태: 대기
생성일: 2026-02-06
신청자: 이순신

품목 목록:
- 인슐린 주사기 × 100개 @ ₩5,000 = ₩500,000
- 수술용 장갑 × 200개 @ ₩2,000 = ₩400,000

총 발주액: ₩900,000
```

### 5️⃣ 물품등록

**목적**: 신규 물품 상세 정보 등록

**등록 절차:**
1. 재고관리 시스템 → **📝 물품등록** 탭
2. **+ 새 물품 등록** 버튼 클릭
3. 정보 입력:

```
제품명: 소독용 알콜 500ml
수량: 200개
단가: ₩15,000
거래처: 약품 B사
유효기간: 2026-08-31
LOT번호: LOT-2025-003
공급내역 보고 대상: ☐ (체크 안 함)
```

4. **물품 등록** 버튼 클릭

**팁:**
- 제품명은 정확하게 입력 (스캔 입고 시 자동 매칭)
- 단가는 세금 제외 금액
- 유효기간이 있는 물품은 반드시 입력
- 의료기기는 "공급내역 보고 대상" 체크

### 6️⃣ 스캔 입고

**목적**: 명세서 또는 바코드 스캔으로 빠른 입고 처리

#### 🔹 바코드 스캔

**사용 시나리오:**
- 개별 물품 입고 시
- 의료진이 현장에서 빠르게 입고 처리

**단계:**

```
1. 재고관리 시스템 → 📱 스캔 입고 탭
   ↓
2. "📱 바코드 스캔" 모드 선택
   ↓
3. "카메라 시작" 버튼 클릭
   ↓
4. 제품 바코드에 카메라 대기
   ↓
5. "촬영" 버튼 클릭
   ↓
6. OCR 인식 완료
   ├─ 제품명 ✓ (자동 인식)
   ├─ 바코드 ✓ (자동 인식)
   ├─ 수량 ✓ (자동 인식)
   ├─ 단가 ✓ (자동 인식)
   ├─ 유효기간 ⚠️ (추가 입력 필요)
   └─ LOT번호 ⚠️ (추가 입력 필요)
   ↓
7. 추가 정보 입력 여부 확인
   ├─ "수정하기" → 필요한 정보 입력
   └─ "그대로 진행" → 바로 입고
   ↓
8. "입고 처리" 버튼 클릭
   ↓
9. ✅ 입고 완료
```

**화면 예시:**
```
┌──────────────────────────────────────────┐

├──────────────────────────────────────────┤

│  │                                  │   │
```

#### 🔹 명세서 스캔

**사용 시나리오:**
- 거래처에서 명세서를 가져왔을 때
- 여러 품목을 한 번에 입고 처리

**단계:**

```
1. 재고관리 시스템 → 📱 스캔 입고 탭
   ↓
2. "📄 명세서 스캔" 모드 선택
   ↓
3. "카메라 시작" 버튼 클릭
   ↓
4. 명세서 전체를 카메라에 대기
   ↓
5. "촬영" 버튼 클릭
   ↓
6. OCR 인식 완료
   ├─ 거래처명 ✓ (자동 인식)
   ├─ 송장일 ✓ (자동 인식)
   └─ 품목 목록 ✓ (자동 인식)
      ├─ 제품명, 수량, 단가
      └─ 유효기간, LOT번호 (필요시 추가)
   ↓
7. 여러 품목 일괄 표시
   ↓
8. 추가 정보 필요 여부 확인
   ↓
9. "입고 처리" 버튼 클릭
   ↓
10. ✅ 모든 품목 일괄 입고 완료
```

**명세서 스캔 결과 예시:**

```
│   유효기간: 2026-06-30                 │
│   LOT: LOT-2025-002                    │
├────────────────────────────────────────┤
│ • 소독용 알콜                           │
```

#### 🔹 파일 업로드

**사용 시나리오:**
- 카메라가 없는 환경
- 이미 촬영된 사진 업로드

## 문제 해결

### ❌ 자주 발생하는 문제

#### 1. 카메라가 인식되지 않음
**증상**: "카메라 접근 권한이 필요합니다" 메시지

**해결 방법:**
1. 브라우저 설정 확인
2. 카메라 권한 허용
3. 다른 앱에서 카메라 사용 중인지 확인
4. 브라우저 재시작

**브라우저별 설정:**

**Chrome:**
- 주소창 왼쪽 🔒 아이콘 클릭
- 카메라 권한 → 허용 선택

**Safari:**
- 설정 → 개인정보 보호 → 카메라 확인
- 해당 사이트 허용

**Firefox:**
- 주소창 왼쪽 🔒 아이콘 클릭
- 카메라 권한 → 허용 선택

---

#### 2. OCR 인식이 제대로 안 됨
**증상**: 스캔 후 데이터가 비어있거나 잘못됨

**원인 및 해결:**

| 원인 | 해결 방법 |
|:---|:---|
| 명세서/바코드 선명도 낮음 | 조명 밝게 하고 다시 촬영 |
| 각도가 틀어짐 | 명세서/바코드를 정면에서 촬영 |
| 카메라 초점이 안 맞음 | 카메라를 천천히 움직여 초점 맞추기 |
| 글씨가 너무 작음 | 카메라를 더 가깝게 대기 |

**팁:**
- 명세서는 A4 크기 전체가 화면에 들어오도록
- 바코드는 중앙에 크게 나오도록
- 조명이 충분한 환경에서 촬영

---

#### 3. 제품이 등록되지 않았다는 오류
**증상**: "등록되지 않은 제품입니다" 메시지

**원인:** 스캔된 제품명이 시스템에 없음

**해결 방법:**
1. 물품등록 탭에서 먼저 물품 등록
2. 제품명을 정확히 입력 (스캔 시 자동 매칭)
3. 다시 스캔 입고 시도

---

#### 4. 발주서가 생성되지 않음
**증상**: "자동 발주 생성" 버튼 클릭해도 반응 없음

**원인:** 안전재고 미달 품목이 없음

**해결 방법:**
1. 현황 대시보드에서 안전재고 미달 품목 확인
2. 없으면 정상 상태 (발주 필요 없음)
3. 수동으로 발주서 생성 필요 시 행정팀에 문의

---

#### 5. 입고 처리 후 재고가 안 늘어남
**증상**: 입고 처리했는데 재고 수량이 변하지 않음

**원인:** 시스템 동기화 지연

**해결 방법:**
1. 페이지 새로고침 (F5 또는 Ctrl+R)
2. 현황 대시보드 다시 확인
3. 여전히 안 되면 행정팀에 문의

---

### 🔧 기술적 문제

#### 데이터베이스 연결 오류
**메시지**: "데이터베이스 연결 실패"

2. 사용자 권한 확인 (행정팀 여부)
3. 행정팀이 아니면 행정팀에 요청

## FAQ

### Q1. 스캔 입고 시 유효기간과 LOT번호는 왜 수동으로 입력해야 하나요?

**A.** OCR 기술의 한계로 명세서에 작은 글씨로 적혀있는 유효기간과 LOT번호를 정확히 인식하기 어렵기 때문입니다. 정확한 정보 입력을 위해 수동 입력이 필요합니다.

---

### Q2. 바코드 스캔과 명세서 스캔의 차이는?

**A.** 
- **바코드 스캔**: 개별 물품 1개 입고 (빠름)
- **명세서 스캔**: 여러 물품 일괄 입고 (효율적)

---

### Q3. 안전재고는 어떻게 설정되나요?

**A.** 물품 등록 시 자동으로 설정됩니다 (현재 수량의 30%). 필요시 행정팀에서 수동으로 조정 가능합니다.

---

### Q4. UDI 보고는 누가 해야 하나요?

**A.** 행정팀에서 월 1회 또는 필요시 진행합니다. 의료기기 공급내역 보고 의무가 있는 경우 필수입니다.

---

### Q5. 거래처를 삭제할 수 있나요?

**A.** 거래처에 연결된 발주서나 입고 이력이 있으면 삭제 불가능합니다. 필요시 행정팀에 문의하세요.

---

### Q6. 재고 조정 시 승인이 필요한가요?

**A.** 네, 손상/분실/폐기 등의 재고 조정은 행정팀 승인이 필요합니다.

---

### Q7. 모바일에서도 사용 가능한가요?

**A.** 네, 모바일 웹 브라우저에서 완벽하게 지원됩니다. 스캔 입고는 모바일에서 더 편합니다.

---

### Q8. 오래된 데이터는 어떻게 삭제하나요?

**A.** 시스템은 모든 데이터를 보관합니다. 법적 보관 기간(3~5년) 후 필요시 행정팀에 문의하세요.

---

### Q9. 발주서를 취소할 수 있나요?

**A.** 상태가 "대기"인 발주서는 취소 가능합니다. "승인" 상태는 행정팀 승인 필요합니다.

---

### Q10. 여러 거래처에서 같은 물품을 구매하는 경우는?

**A.** 물품명은 같지만 거래처는 다르게 등록하면 됩니다. 스캔 입고 시 거래처 정보도 함께 기록됩니다.

## 7. 직원 권한 설정 가이드

관리자는 **'시스템설정 > 직원별 권한 제어'** 메뉴를 통해 각 직원의 접근 권한을 세밀하게 관리할 수 있습니다.

1.  **권한 제어 항목**:
    - **재고관리 접근**: 재고 현황 확인, 입출고, UDI 보고 등 재고 메뉴 활성화 여부
    - **인사관리 접근**: 직원 명단 조회, 급여, 근태 등 인사 메뉴 활성화 여부
    - **전자결재 사용**: 기안 작성 및 결재 승인 프로세스 참여 권한
    - **관리자메뉴 접근**: 시스템 설정, 권한 제어, 데이터 초기화 메뉴 활성화 여부

2.  **설정 방법**:
    - 왼쪽 직원 명단에서 대상을 선택합니다.
    - 오른쪽 권한 스위치를 클릭하여 활성화(파란색) 또는 비활성화(회색)합니다.
    - 설정은 즉시 저장되며, 해당 직원이 시스템 이용 시 실시간으로 메뉴가 나타나거나 사라집니다.

3.  **주의사항**:
    - 병원장(사번 1번) 계정은 시스템 보호를 위해 항상 모든 권한이 유지되는 것을 권장합니다.
    - 행정팀 직원은 업무를 위해 '인사관리'와 '재고관리' 권한이 필수적으로 요구됩니다.
