# 병원 ERP 시스템 - 고급 기능 구현 가이드

## 📋 개요
이 문서는 병원 ERP 시스템에 추가된 고급 기능들의 구현 방법과 사용 방법을 설명합니다.

---

## 1️⃣ 급여 자동 이체 (급여자동이체.tsx)

### 기능 설명
직원들의 급여를 은행에 자동으로 이체하는 시스템입니다.

### 주요 기능
- ✅ 월별 급여 정산 데이터 조회
- ✅ 계좌번호, 은행명, 예금주명 자동 입력
- ✅ 공제액 자동 계산
- ✅ 이체액 실시간 계산
- ✅ CSV 파일 다운로드
- ✅ 은행 API 연동 (실제 이체)

### 사용 방법
1. 인사관리 → 급여명세 → "💳 급여 자동 이체" 탭 클릭
2. 정산 월 선택
3. 이체 대상 직원 확인
4. "💳 이체 실행" 버튼 클릭
5. 확인 모달에서 "이체 실행" 재확인

### 데이터베이스 필드
```sql
-- payroll 테이블에 추가 필드
ALTER TABLE payroll ADD COLUMN IF NOT EXISTS (
    bank_name VARCHAR(50),
    account_number VARCHAR(50),
    account_holder VARCHAR(50),
    transfer_status VARCHAR(20),
    transfer_date TIMESTAMP,
    transfer_payload JSONB
);
```

## 2️⃣ 세금 신고 자동화 (세금자동신고.tsx)

### 기능 설명
4대보험료와 소득세를 자동으로 계산하고 신고하는 시스템입니다.

### 자동 계산 항목
- 건강보험: 3.395%
- 장기요양보험: 0.775%
- 고용보험: 0.8%
- 국민연금: 4.5%
- 소득세: 과세표준에 따라 6~35%

### 사용 방법
1. 인사관리 → 급여명세 → "🏛️ 세금 신고" 탭 클릭
2. 신고 년도 선택
3. 세금 신고 현황 확인
4. "📤 신고 제출" 버튼 클릭
5. 세무서 시스템에 자동 제출

### 신고 프로세스
```
월별 급여 데이터 수집
    ↓
직원별 연간 급여 집계
    ↓
4대보험료 자동 계산
    ↓
소득세 자동 계산
    ↓
신고 데이터 생성
    ↓
세무서 API 제출
```

## 3️⃣ 연말정산 (연말정산.tsx)

### 기능 설명
연간 급여를 기반으로 연말정산을 자동으로 계산합니다.

### 계산 항목
- 기본공제: ₩1,500,000
- 과세표준: 연간급여 - 기본공제
- 산출세액: 과세표준 × 세율
- 환급/추가납부: 기납부세액 - 산출세액

### 사용 방법
1. 인사관리 → 급여명세 → "📊 연말정산" 탭 클릭
2. 정산 년도 선택
3. 직원별 정산 현황 확인
4. "보기" 버튼으로 원천징수영수증 확인
5. "📧 이메일 발송" 또는 "📥 다운로드"

### 원천징수영수증 자동 발급
- 직원 정보 자동 입력
- 소득 정보 자동 계산
- 세금 정보 자동 기입
- 보험료 정보 자동 계산
- 이메일 자동 발송 또는 파일 다운로드

## 4️⃣ 업체별 물품 현황 (업체별현황.tsx)

### 기능 설명
업체별 구매 현황과 환자 처방 금액을 관리합니다.

### 주요 기능

#### A. 업체별 구매 현황
- 업체별 총 구매액
- 월별 거래 현황
- 품목별 상세 내역
- CSV 다운로드

#### B. 환자 처방 금액 자동 분류
- 엑셀 파일 업로드
- OCR 인식 (품목명, 수량, 단가)
- 자동 분류 및 저장
- 환자별 처방액 집계

### 사용 방법

**업체별 현황 조회:**
1. 재고관리 → "🏢 업체별 현황" 탭
2. 조회 월 선택
3. 업체별 구매 현황 확인

**환자 처방 데이터 업로드:**
1. "📤 엑셀 업로드" 버튼 클릭
2. 파일 선택 (CSV 또는 XLSX)
3. 파일 형식: 환자명, 환자ID, 품목명, 수량, 단가
4. 업로드 완료 시 자동 분류

### 엑셀 파일 형식
```
환자명,환자ID,품목명,수량,단가
김철수,P001,항생제,1,50000
이영희,P002,밴드,10,5000
박민준,P003,주사기,5,3000
```

## 5️⃣ 유효기간 알림 (유효기간알림.tsx)

### 기능 설명
제품의 유효기간을 추적하고 임박/만료 제품을 자동으로 알립니다.

### 알림 기준
- **6개월 이내**: 행정팀에 경고 알림
- **30일 이내**: 긴급 알림
- **만료됨**: 즉시 폐기 알림

### 자동 알림 대상
- 행정팀 모든 직원
- 관리자

### 사용 방법
1. 재고관리 → "⏰ 유효기간 알림" 탭
2. 임박 제품 및 만료 제품 확인
3. 필요시 "폐기 처리" 버튼 클릭
4. "📥 보고서 다운로드"로 현황 보고

### 알림 설정
- 자동 확인: 매일 오전 8시
- 알림 대상: 행정팀
- 알림 채널: 시스템 푸시 알림

## 🔧 Supabase 필수 설정

### 테이블 생성
```sql
-- 급여 이체 기록
CREATE TABLE IF NOT EXISTS salary_transfers (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    staff_id UUID REFERENCES staffs(id),
    month VARCHAR(7),
    bank_name VARCHAR(50),
    account_number VARCHAR(50),
    transfer_amount DECIMAL(10, 2),
    transfer_date TIMESTAMP,
    status VARCHAR(20),
    created_at TIMESTAMP DEFAULT NOW()
);

-- 세금 신고 기록
CREATE TABLE IF NOT EXISTS tax_reports (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    year INTEGER,
    data JSONB,
    status VARCHAR(20),
    report_date TIMESTAMP,
    created_at TIMESTAMP DEFAULT NOW()
);

-- 환자 처방 데이터
CREATE TABLE IF NOT EXISTS patient_prescriptions (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    patient_name VARCHAR(100),
    patient_id VARCHAR(50),
    item_name VARCHAR(100),
    quantity INTEGER,
    unit_price DECIMAL(10, 2),
    total_amount DECIMAL(10, 2),
    prescription_date DATE,
    status VARCHAR(20),
    created_at TIMESTAMP DEFAULT NOW()
);

-- 유효기간 추적
CREATE TABLE IF NOT EXISTS expiration_tracking (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    item_id UUID REFERENCES inventory(id),
    expiration_date DATE,
    alert_sent BOOLEAN DEFAULT FALSE,
    alert_date TIMESTAMP,
    created_at TIMESTAMP DEFAULT NOW()
);
```

### 인덱스 생성
```sql
CREATE INDEX idx_salary_transfers_month ON salary_transfers(month);
CREATE INDEX idx_tax_reports_year ON tax_reports(year);
CREATE INDEX idx_patient_prescriptions_date ON patient_prescriptions(prescription_date);
CREATE INDEX idx_expiration_tracking_date ON expiration_tracking(expiration_date);
```

## 📊 통합 워크플로우

### 월별 급여 처리 프로세스
```
1. 근태 데이터 수집 (매일)
   ↓
2. 월말 급여 계산 (매월 25일)
   ↓
3. 세금/보험료 자동 계산
   ↓
4. 연말정산 데이터 준비 (필요시)
   ↓
5. 급여 이체 (매월 26일)
   ↓
6. 세금 신고 (자동)
   ↓
7. 원천징수영수증 발급 (연말)
```

### 재고 관리 프로세스
```
1. 물품 입고 (엑셀 업로드 또는 스캔)
   ↓
2. 환자 처방 데이터 분류 (자동)
   ↓
3. 업체별 현황 분석 (월별)
   ↓
4. 유효기간 추적 (매일 자동)
   ↓
5. 임박/만료 제품 알림 (자동)
   ↓
6. 폐기 처리 (수동)
```
